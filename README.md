## 📖 Project Overview

**MythicLedger** is a robust .NET 8 Web API designed to power a modern digital products storefront. Built to handle the specific transactional needs of virtual goods e-commerce—such as processing orders for gaming assets (like Mobile Legends diamonds) and managing software subscriptions—this API serves as the secure, high-performance foundation for a Next.js frontend application.

## 🎯 Core Purpose

The primary goal of this project was to architect a scalable, enterprise-grade backend capable of securely managing customer ledgers, tracking complex order life-cycles, and handling secure shop administration. It acts as the central source of truth for the business, ensuring data integrity across all user transactions and administrative actions.

## 🏗️ Engineering & Architecture Focus

To ensure the codebase remains maintainable and testable as the business scales, this API was built using strict **Clean Architecture** principles. 

By heavily decoupling the system's layers, the core business rules are completely isolated from external frameworks and database concerns.

*   **Domain Layer:** Encapsulates the core enterprise logic, rich entities (`User`, `Customer`, `Order`), and strictly typed custom Enums.
*   **Application Layer:** Orchestrates business use cases, handles DTO transformations, and defines contract interfaces (e.g., `IAuthService`, `IUserRepository`).
*   **Infrastructure Layer:** Manages all data access, implements repository patterns, and configures the database context.
*   **API / Presentation Layer:** Handles HTTP requests, JWT validation, middleware, and Swagger documentation.

## 🌟 Technical Highlights

*   **Complex ORM Bridging:** Successfully engineered a bridge between a legacy database schema (originally generated by **Prisma** in a Node.js ecosystem) and **Entity Framework Core**. This required advanced Npgsql 8.0 configurations to seamlessly map native PostgreSQL custom enum types to C# models.
*   **Secure Authentication:** Implemented robust JWT (JSON Web Token) bearer authentication, featuring strict issuer/audience validation and secure password hashing.
*   **Role-Based Access Control (RBAC):** Built-in support for hierarchical endpoint authorization, strictly separating capabilities between `SUPER_ADMIN` and `SHOP_ADMIN` roles.
*   **Dependency Injection:** Extensively utilized built-in IoC containers for scoping services and repositories, ensuring highly testable and modular code.
